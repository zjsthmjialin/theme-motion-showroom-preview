MemWeaverObsidian x Codex

Long-term memory for AI work

给 Codex 一块可同步的长期记忆硬盘

用 Obsidian 知识库管理 CodeX 记忆，用全局 AGENTS.md 让 Codex 每次任务自动读取、任务结束自动写回，维持长期记忆。

纯 Markdown零部署Mac / Windows可分发
01Problem

AI forgets between tasks

每个新任务都像重新认识你

重复交代

偏好、项目状态、边界条件反复说明，时间被消耗在“重新同步背景”。

经验流失

一次任务里的判断和踩坑，如果不沉淀，下次仍然可能重新犯。

上下文过载

把所有材料塞进提示词会变慢、变乱，也更容易污染当前任务。

02Solution

One vault, three memory zones

一份 Obsidian 库，三层记忆入口

AI-Memory

偏好、项目索引、总索引

10-Cards

可复用结论卡

20-Projects

项目状态与决策

Codex 先读入口，再按任务相关性读取少量文件；默认不通读全库。

03Workflow

Read, work, write back

任务开始自动读取，任务结束自动沉淀

Start

扫描入口

列出 AI-Memory/，读取相关偏好与项目状态。

Work

按需检索

需要历史经验时检索 10-Cards/，最多沿双链延伸一层。

End

只追加写回

只记录未来可复用的结论，并重建 INDEX.md。

04Distribution

One package, local paths

一份知识库包，同时支持 Mac 和 Windows

Mac

~/.codex/AGENTS.md 指向本机 Vault 路径。

JS-MemWeaver-Vault

同步盘中只保留一份内容库

Windows

C:\Users\...\.codex\AGENTS.md 指向本机 Vault 路径。

内容库不分叉；每台电脑只配置自己的全局 AGENTS.md。

05Start

From blank memory to working memory

拿到压缩包，三步开始使用

1

用 Obsidian 打开 JS-MemWeaver-Vault。

2

把 Mac 或 Windows 提示词发给本机 Codex。

3

新开任务测试自动读取，并开始沉淀自己的偏好和经验。

MemWeaver 的目标不是替你记住一切，而是让真正会复用的经验，有地方留下来。
